Introduction

Smart Learning is an innovative Android application designed to help students manage their academic plans and monitor their learning performance systematically. The application integrates local storage (Room) and cloud services (Firebase & Retrofit) to provide fast data access while delivering daily learning tips to motivate users. In addition, it utilizes device sensors to help ensure a conducive learning environment for users.

SDG Alignment

SDG 4: Quality Education - This application empowers students with digital tools to independently organize their curriculum, track academic progress through quizzes, and conveniently access educational tips that promote lifelong learning.

Laboratory Work

Lab 1: UI Design

Exploration of user interface layouts using declarative UI mechanisms.

This lab helped me understand the fundamentals of Android user interface design using Jetpack Compose. I learned how to create visually appealing layouts and organize UI components effectively to improve user experience. The knowledge gained from this lab formed the foundation for developing the Smart Learning application.
Lab 2: Interaction

Implementing dynamic element triggers and active interactive states.

Through this lab, I learned how to implement user interactions such as button clicks, input handling, and state management in Android applications. These features enabled the application to respond dynamically to user actions. The skills acquired were useful for creating interactive learning and quiz functionalities in the project.
Lab 3: Material Design

Applying modern design principles to provide standardized presentation.

This lab introduced the principles of Material Design and how they can be applied to create consistent and modern user interfaces. I learned to use Material components, themes, and layouts to enhance usability and visual appeal. Applying these concepts improved the overall look and feel of the Smart Learning application.
Lab 4: Navigation

Configuring backstacks and single-activity navigation flows between screens.

In this lab, I learned how to implement navigation between multiple screens using the Navigation Component in Jetpack Compose. This knowledge allowed me to create a seamless user experience when moving between different sections of the application. The navigation structure became an important part of organizing the app's features effectively.
Lab 5: Local Database

Implementing architecture-ready offline data structures and entities.

This lab provided practical experience in using Room Database for local data storage and management. I learned how to create entities, data access objects (DAOs), and database operations to persist user data efficiently. The implementation of Room was essential for storing academic plans and quiz history within the Smart Learning application.

Development Projects

Project 1: Smart Learning (Core Features)
Problem Statement: Many students struggle to manage their learning activities efficiently due to the lack of a centralized platform for educational resources, self-assessment, and progress tracking. As a result, students often find it difficult to monitor their academic performance, review past learning outcomes, and stay motivated throughout their learning journey.
Lessons Learned: Through the development of Smart Learning, I learned how to design and implement a multi-screen Android application using Jetpack Compose and Navigation. I gained hands-on experience in creating educational features such as quiz management, score tracking, quiz history recording, and resource searching while ensuring smooth navigation between screens. This project also enhanced my understanding of user-centered design, application architecture, and the importance of developing digital tools that support SDG 4: Quality Education.

Project 2: Smart Learning (Advanced Features)
Problem Statement: Students often struggle to maintain a structured study plan and stay motivated while lacking a centralized platform that tracks both local progress and community achievements. Additionally, many learners are unaware of environmental factors, such as poor lighting conditions, which can negatively affect their focus and eye health during long study sessions.
Lessons Learned: Through this project, I mastered the integration of a "single source of truth" by combining local data persistence via Room with real-time cloud synchronization using Firebase and Retrofit. I also gained valuable experience in utilizing hardware sensors to improve user well-being and learned how to implement the MVVM architecture to manage complex states in a modern Jetpack Compose application.

Resources & Credits

APIs & Libraries Used

  • Jetpack Compose Suite: For building native modern layouts declaratively.
  • Room Persistence Library: Local relational mapping abstraction over SQLite.
  • Retrofit (Square Inc.): Type-safe HTTP client abstraction for loading learning tips.
  • Firebase Android SDK: Handling cloud storage and remote data streams.
  • Android Hardware Sensor API: Managing hardware inputs for ambient environmental metrics.
